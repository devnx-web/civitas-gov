<feature>
  <name>canonicalJSON determinístico + hash chain SHA-256 da auditoria</name>
  <files>src/lib/auditoria-hash.ts, src/lib/auditoria-hash.test.ts</files>
  <behavior>
    canonicalJSON(obj): produz string canônica determinística.
    - Caso 1: chaves de objeto sempre em ordem alfabética — `canonicalJSON({b:1,a:2})` === `canonicalJSON({a:2,b:1})`.
    - Caso 2: `Date` → string ISO 8601 UTC truncada (mesma precisão sempre, ex. milissegundos) — mesmo Date em fusos diferentes gera a mesma string.
    - Caso 3: `Decimal` do Prisma → string com precisão fixa (não float) — `1.10` e `1.1` normalizam para a mesma representação.
    - Caso 4: chaves com valor `undefined` são omitidas; `null` é preservado como `null`.
    - Caso 5: sem trailing whitespace, sem espaços entre tokens (JSON compacto).
    - Caso 6: objetos aninhados e arrays recursam mantendo a ordenação; ordem de array é preservada (arrays são ordenados pela posição, não reordenados).
    - Caso 7: `NaN` e `Infinity` lançam erro explícito (não podem entrar na cadeia).
    - Caso 8: timestamps próximos ao boundary de segundo/dia normalizam corretamente.
    - 30+ casos no total cobrindo os 8 grupos acima, com `toMatchSnapshot()` em fixtures representativas — qualquer drift quebra o build.

    computarHash(row, prevHash): retorna o hex SHA-256.
    - Determinístico: chamar duas vezes com a mesma entrada retorna o mesmo hash.
    - O hash incorpora prevHash — mudar prevHash muda o resultado.
    - Para o primeiro registro de um tenant (prevHash null/vazio), usa uma string-semente fixa documentada.

    verificarCadeia(tenantId, periodo?): recomputa a cadeia em ordem [tenantId, criadoEm asc].
    - Cadeia íntegra → retorna `{ integra: true }`.
    - Registro adulterado (antes/depois alterado) → retorna `{ integra: false, registroDivergente: id }` apontando o PRIMEIRO divergente.
    - prevHash quebrado (não bate com o currentHash anterior) → detectado.

  </behavior>
